# 🎬 ClientScriptRunner - مدير السكريبتات المتقدم

## نبذة عن البرنامج
**ClientScriptRunner** هو plugin متطور لنظام Enigma2 يوفر واجهة حديثة واحترافية لتشغيل واستعراض والتحكم بـ السكريبتات على أجهزة VU+ Solo2.

### الميزات الرئيسية ✨
- ✅ **تشغيل في الخلفية**: تشغيل السكريبتات في الصمت بدون تعطيل الجهاز
- ✅ **واجهة مزدوجة الجانب**: استعراض ملفات + قائمة عمليات مشغلة
- ✅ **تحكم كامل**: إيقاف فردي أو جماعي للعمليات
- ✅ **دعم ملفات متعددة**: Python (.py) و Shell (.sh)
- ✅ **تصميم عصري**: واجهة جميلة وسهلة الاستخدام
- ✅ **معلومات حية**: عرض PID والأوقات والحالة

## 📝 أمثلة على السكريبتات

### مثال Python (.py):
```python
#!/usr/bin/env python3
import time
import subprocess

while True:
    # القيام بعملية معينة
    print("[ClientScriptRunner] تم التشغيل")
    subprocess.call(['echo', 'Script is running...'])
    time.sleep(5)
```

### مثال Shell (.sh):
```bash
#!/bin/bash
# تشغيل نسخة احتياطية
while true; do
    echo "[ClientScriptRunner] نسخة احتياطية: $(date)"
    sleep 10
done
```

## ⚙️ الإعدادات

### تغيير مسار الملفات الافتراضي:
عدّل في `main.py`:
```python
self.file_explorer = FileExplorer("/media/hdd/")  # غيّر المسار هنا
```

### الملفات المدعومة:
حالياً: `.py` و `.sh`

لإضافة صيغ أخرى، عدّل في `FileExplorer`:
```python
self.supported_extensions = ['.py', '.sh', '.bash']  # أضف هنا
```

## 🐛 استكشاف الأخطاء

### المشكلة: السكريبت لا يعمل
**الحل:**
- تأكد من أن الملف قابل للتنفيذ: `chmod +x script.py`
- تحقق من رأس الملف: `#!/usr/bin/env python3`
- تحقق من وجود الملف في المسار الصحيح

### المشكلة: لا تظهر الملفات
**الحل:**
- تحقق من المسار الافتراضي
- تأكد من صلاحيات القراءة: `chmod 755 /media/hdd/`

### المشكلة: العمليات لا تُظهر
**الحل:**
- اضغط الزر الأخضر لتحديث الشاشة
- تحقق من أن السكريبتات تعمل فعلاً
